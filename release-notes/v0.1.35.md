## Changed
- Servers create flow is now dependency-first: Datacenter -> Rack -> Switch -> Switch Port.
- Add Server form no longer loads every port in advance; it fetches only ports of the selected switch.
- Port selector labels now show interface name, port description, and live link state (Connected / Not Connected / SFP Absent).

## Added
- New internal API endpoint `switch/ports` for switch-scoped port listing with datacenter validation.
- New per-server inline edit panel in Servers list for post-create mapping updates.
- Post-create mapping now supports updating switch/port binding and PRTG traffic sensors from each server row.

## Fixed
- Prevented cross-datacenter switch assignment in server mapping updates.
- Reduced add-server form complexity to avoid accidental wrong mappings when additional datacenters/switches are added.
