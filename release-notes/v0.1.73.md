# v0.1.73

### Changed
- Servers table no longer uses inline row edit expansion.
- Added clear per-row actions: `View`, `Edit`, `Delete`.
- Added a unified server details workspace with mode switch:
  - `View` for read-only details.
  - `Edit` for full update flow.
- Reorganized server details into structured sections:
  - Overview
  - Network Control (single SNMP control port mapping)
  - Traffic links + aggregated counters
  - Hardware/Traffic sensor sections
  - Public IP discovery checks
  - WHMCS discovery status/log visibility
  - iLO controls block

### Added
- Schema migration `v14` for `sensor_type` in `mod_dcmanage_server_traffic_sensors`.
- Explicit control-port persistence for server block/unblock target (`action_switch_id` / `action_port_id`).

### Fixed
- Improved servers details layout consistency (spacing/alignment/buttons).
- Monitoring row parser now supports typed sensor rows with safer dedupe behavior.
