## Changed
- Servers create/edit forms now use full dependency chain (`Datacenter -> Rack -> Switch -> Port`) in both create and edit flows.
- Server edit now saves core server fields (`datacenter`, `rack`, `hostname`, `U start/height`, `notes`, `iLO`) together with link mappings in one place.
- Traffic links are now row-based (`Switch + Port`) so each server can map multiple traffic ports.

## Added
- Monitoring links are now row-based with `PRTG instance + sensor + alert action`.
- Alert action options were added for monitoring rows: `None`, `Email`, `SMS`, `Email + SMS`, `Ticket`.
- New schema migration `v9` adds `alert_action` to monitoring sensor storage.
- Persian/English-aware port search was added to server port selector inputs.

## Fixed
- Datacenter ownership is validated for selected rack/switch/port during save.
- Legacy single-link/sensor payloads remain backward-compatible via fallback parsing.
