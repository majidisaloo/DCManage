# v0.1.47

## Changed
- Update flow moved to queue-based execution to avoid task collisions and overlapping update attempts.
- Dashboard update panel now polls runtime update status and supports cancel requests.

## Added
- Added `update/status` and `update/cancel` API endpoints.
- Added release-package completeness validation (required module files + version consistency + content-length validation).

## Fixed
- Fixed false-positive `updated` log entries; success is logged only after copy + installed version verification.
- Increased update request timeouts and added retries for unstable network/release fetch conditions.
