# v0.1.70

## Changed
- Monitoring top action is now `Add Monitoring Instance` to match multi-platform design.
- Discovery workflow was moved from Monitoring tab into Server edit context.
- PRTG probe parsing now removes internal/numeric pseudo probe rows.

## Added
- Monitoring `View` shortcut in instances list.
- Monitoring Group Mapping panel with per-instance/per-switch mappings.
- New mapping purposes: `Traffic`, `Hardware`, `Public Monitoring`, `Client Discovery`.
- New DB table: `mod_dcmanage_monitoring_group_map`.
- Per-server Discovery panel for ping/common-port precheck.

## Fixed
- Probe dropdown no longer floods with internal numeric objects in common PRTG layouts.
