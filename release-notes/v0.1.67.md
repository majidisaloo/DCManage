## Changed
- Monitoring management is now unified for multiple platform types in one list/add flow (`PRTG`, `SolarWinds`, `Cacti`).
- Server traffic and monitoring row selectors now use embedded search inputs inside the same selector block for cleaner operator workflow.
- PRTG instance auth now supports both `Username + Passhash` and `API Token` modes.

## Added
- New PRTG hierarchy APIs:
  - `prtg/probes`
  - `prtg/groups`
  - `prtg/devices`
  - `prtg/device-sensors`
- New Monitoring tab browser UI to walk `Instance -> Probe -> Group -> Sub Group -> Device -> Sensors`.
- New discovery API/UI for host and common TCP port checks (`monitoring/discover`) without shell execution.
- New schema migration `v10` adding monitoring instance fields: `type`, `auth_mode`.

## Fixed
- Improved PRTG 401 handling with auth fallback sequence (`username+passhash`, `username+apitoken`, `apitoken`).
- Strengthened datacenter-scoped option filtering so rack/switch selectors are rebuilt per selected datacenter.
