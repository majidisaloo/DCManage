## Added
- Added `Test Mode` setting to simulate quota breach flow without real suspend/enforce actions.

## Changed
- Monitoring instance create now supports multi-platform records (`PRTG`, `SolarWinds`, `Cacti`) without forcing secret for non-PRTG providers.
- PRTG credential field clarified to accept `passhash / apitoken`.
- PRTG client now normalizes base URL (trailing `index.htm` / `home` removed).
- PRTG auth fallback chain expanded (`username+passhash`, `username+password`, `username+apitoken`, `apitoken`).
- Prefixed secrets like `apitoken=...` and `passhash=...` are now normalized automatically.

## Fixed
- Reduced common `PRTG HTTP 401` scenarios in self-signed and mixed-auth setups.
- Usage engine now skips actual enforce queue actions while Test Mode is enabled and writes explicit test-mode logs.
