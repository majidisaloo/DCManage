## Changed
- Dashboard Queue KPI now opens a dedicated `Queue` tab instead of `Settings`.
- Dashboard KPI icons are larger and better aligned with labels for wide-screen readability.
- Updater network timings are tuned for faster completion (lower timeout/retry overhead).

## Added
- New `Queue` tab with job listing, search, status filter, and pagination.
- Queue controls for operations: `Cancel` (pending/running), `Retry` (failed/canceled), and `Clear Done/Failed`.
- Cached latest-release metadata fallback for resilient version checks when GitHub API is temporarily unavailable.

## Fixed
- Fixed `dashboard/version` 400 failures by returning safe version payload even when remote release fetch fails.
- Fixed updater cancel-stuck behavior by clearing orphan cancel flags and auto-cleaning stale update jobs.
- Reduced false/stuck update UX states by kicking one immediate queue batch on apply (cron remains fallback).
