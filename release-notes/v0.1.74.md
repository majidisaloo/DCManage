# v0.1.74

### Changed
- Monitoring instance actions now keep context on the Monitoring tab and preserve the active `View` modal state after create/delete mapping actions.
- Monitoring labels were normalized to vendor-agnostic wording (`Monitoring URL`, `Monitoring Username`, `Monitoring Secret`) while keeping PRTG-specific auth behavior.
- Modal styling was unified across Monitoring, Servers, and Packages for consistent spacing and action alignment.

### Added
- Added focused loading UX for monitoring hierarchy mapping forms (row-level busy state + spinner).

### Fixed
- Fixed Server edit selector escaping that caused Safari/WebKit `SyntaxError: Invalid escape in identifier`.
- Improved PRTG hierarchy filtering to reduce invalid/numeric/internal probe/group/device rows and de-duplicate noisy results.
